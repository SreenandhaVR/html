Web Forms

Notes on HTML forms — controls, syntax, and input types

Web forms are one of the main ways users interact with websites. When we fill out a quiz, sign up for a newsletter, or enter payment details to make an online purchase, we are using forms to input data.

Forms can be basic and contain a couple of fields, like a name or email for a newsletter subscription. They can also be complex, such as online shopping forms that require a mailing address, credit card number, billing address, and delivery address.

A form's complexity depends on how much information needs to be collected.

While most forms require processing by a web server through server-side programming, the user interface elements — the parts we see in the browser, like text fields and submit buttons — are created with HTML.

On web development teams, front-end developers typically create the interface elements, while back-end developers manage the functionality and data processing.

There are also many services for creating forms, such as Typeform or Google Forms.

Form Controls

Elements that allow users to input data and interact with web forms (such as text fields, checkboxes, and buttons) are called form controls. Each form control has a specific role and purpose.

Form Syntax

Start with the <form> tag and end with the </form> tag. A form has two key attributes:

  • action — where the data is sent for processing; must be a relative or absolute URL.
  • method — how the form data is sent; GET or POST.
<form action="" method="">

</form>

GET vs. POST

GET sends data as part of the URL. Ideal for bookmarking and sharing.

Example: https://store.com/search?q=black+pants

POST sends data in the HTTP request body and keeps it hidden. It is more secure and ideal for sensitive information and large amounts of data.

The <input> Element

The <input> element is used for different types of data.

Include the name attribute to add a reference to the data once the form is submitted.

<form action="" method="">
  <input name="email">
  <input name="password">
</form>

The <label> Element

Used to create a caption for a form control.

<form action="" method="">
  <label>Email</label>
  <input name="email">
  <label>Password</label>
  <input name="password">
</form>

The <button> Element

Sends the form data to the server for processing. The button text is added between the tags.

<form action="" method="">
  <label>Email</label>
  <input name="email">
  <label>Password</label>
  <input name="password">
  <button>Create account</button>
</form>

Structuring Forms

<form action="" method="">
  <h2>Create an account</h2>
  <div>
    <label>Email</label>
  </div>
  <div>
    <label>Password</label>
  </div>
  <button>Create account</button>
</form>

Forms also have specific structural elements that add additional organization:

  • fieldset — groups related form controls together.
  • legend — provides a caption or title for the fieldset.
<form action="" method="">
  <fieldset>
    <legend>Create an account</legend>
    <div>
      <label>Email</label>
      <input name="email">
    </div>
    <div>
      <label>Password</label>
      <input name="password">
    </div>
    <button>Create account</button>
  </fieldset>
</form>

Implicit <label> Association

<form action="" method="">
  <label>Email
    <input name="email">
  </label>
  <label>Password
    <input name="password">
  </label>
  <button>Create account</button>
</form>

The input is nested inside the label — no id/for pairing needed.

Explicit <label> Association

<form action="" method="">
  <h2>Create an account</h2>
  <div>
    <label for="email">Email</label>
    <input id="email" name="email">
  </div>
  <div>
    <label for="password">Password</label>
    <input id="password" name="password">
  </div>
  <div>
    <button>Create account</button>
  </div>
</form>

The for attribute on the label matches the id on the input.

Input Types

Input elements handle different types of data. Adding the type attribute specifies each input's data type, which determines how the input functions. If no type is specified, the default is text — a single-line text field.

This is appropriate for something like a name or username.

<input type="text" name="name">
<input type="text" name="username">

Email Type

Creates a single-line text field used specifically for email addresses. Unlike text, which accepts any characters, email includes built-in form validation. It checks that the format matches name@domain or name@domain.tld — it does not verify that the address actually exists.

TLD (Top-Level Domain)

The last part of a domain name. Examples: .com, .net, .org, .edu, .gov.

  • www.website.com
  • www.website.org
  • www.website.edu

Password Type

Looks like a single-line text field, but the value is obscured as you type, displayed as asterisks or dots depending on the browser and operating system. 

<input type="password" name="password">

Submit Type

Setting type="submit" renders a button that sends the form data to the server. The default button text is "Submit" — change it with the value attribute. 

<input type="submit" value="Create account">

Putting It Together

<form action="" method="">
  <h2>Create an account</h2>
  <input type="email" name="email">
  <input type="password" name="password">
  <input type="submit" value="Create account">
</form>

<input> vs. <buttn>

<input> is used for form data, <button> is used for actions

<input type="submit" value="Create account">
<button> type="submit">
